EV Diagnostic Assistant

AI-Powered Repair Manual Search System

An Intelligent Solution for EV Technicians

Internship Project | April 2026

Introduction

Problem Statement

EV technicians waste valuable time manually searching through repair manuals to find diagnostic answers, leading to delays, errors, and inefficiency in vehicle troubleshooting.

Our Solution

We built an AI-powered web application that instantly answers diagnostic questions based on uploaded EV repair manuals using Retrieval-Augmented Generation (RAG) technology.

Key Benefit

Technicians can now ask natural language questions and receive accurate, manual-based answers with citations in seconds instead of minutes.

Features Built

Backend Optimization

We reduced the backend memory footprint from 500MB to 100KB by replacing heavy machine learning models with the lightweight BM25 algorithm, enabling deployment on free tier infrastructure.

Audio Transcription

We implemented hands-free voice input by integrating the Groq API for real-time audio transcription, allowing technicians to dictate diagnostic questions while working on vehicles.

Data Persistence & Documentation

We fixed user account persistence issues by configuring persistent storage on Render and created 80+ pages of comprehensive technical documentation for the project.

Tools & Technologies Used

Frontend Stack

We built the user interface using React with Vite for fast development, Tailwind CSS for responsive styling, and the Web Audio API for browser-based audio recording.

Backend Stack

The backend is powered by Flask REST API with Python, SQLite for persistent data storage, and PyPDF2 for reliable PDF processing without system dependencies.

AI & Search Technology

We implemented the BM25 algorithm for lightweight keyword-based search, RAG pipeline for context-aware responses, and cross-encoder re-ranking for improved result quality.

Deployment & Integration

The system is deployed on Render for the backend and Vercel for the frontend, with Groq API integration for audio transcription and GitHub for version control with secret scanning.

Sample Demonstration

💬 Chat Interface

Technicians ask diagnostic questions in natural language and receive instant answers directly from uploaded manuals with source citations for verification.

📤 Manual Upload & Processing

Users upload PDF repair manuals which are automatically parsed, chunked, and indexed using BM25 for fast and accurate retrieval during searches.

🎤 Voice Input Feature

Technicians can use hands-free voice input by recording audio directly in the browser, which is transcribed using Groq API and automatically inserted into the chat.

📋 Chat History Management

All conversations are automatically saved with optimistic persistence, allowing technicians to review past diagnostics and rename conversations for better organization.

🌙 User Preferences

The application includes a dark mode toggle and persistent user settings, providing a comfortable interface for technicians working in various lighting conditions.
