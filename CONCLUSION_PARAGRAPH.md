We created a complete system from scratch. The frontend is built with React. The backend uses Flask. The database is SQLite. We optimized everything to run on free cloud infrastructure. We reduced memory usage by 99.98%. We implemented seven major features. We fixed six critical issues. We created eighty pages of documentation. We worked for thirty-two hours. Everything is production-ready.

## Key Features

The system has a chat interface. Technicians ask questions here. They get instant answers. The answers include citations. Users can upload PDF manuals. The system processes them automatically. Users can use voice input. They can speak their questions. The system transcribes the audio. Chat history is saved automatically. Users can rename conversations. Users can delete old chats. Dark mode is available. Users can toggle between light and dark themes.

## Technical Achievements

We optimized the backend successfully. We replaced heavy ML models with BM25. This reduced memory from 500MB to 100KB. We integrated the Groq API. This enables audio transcription. We fixed CORS configuration. Frontend and backend communicate seamlessly. We solved database persistence issues. User accounts survive redeployment. We implemented graceful error handling. The system starts even if infrastructure fails. We migrated from PyMuPDF to PyPDF2. This works on all platforms. We fixed UI overflow issues. The interface looks professional on all devices.
