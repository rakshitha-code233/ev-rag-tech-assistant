# Organization / Platform Details - 4 Key Points

## 1. Frontend Architecture & Technology Stack

The frontend is built as a modern Single Page Application (SPA) using React with Vite as the build tool. We use Tailwind CSS for responsive, utility-first styling that ensures the application looks professional and functions seamlessly across all devices including desktops, tablets, and mobile phones. The application implements client-side routing using React Router, allowing users to navigate between different pages (Chat, History, Upload, Dashboard) without full page reloads, providing a smooth and responsive user experience. State management is handled through React Context API, which manages authentication state (user login information), theme preferences (dark/light mode), and chat state (current conversation and messages). The frontend communicates with the backend through REST API calls using the Fetch API, with proper error handling and loading states. The application includes accessibility features like semantic HTML, ARIA labels, and keyboard navigation support. The build process is optimized for production, with code splitting, lazy loading, and minification to ensure fast load times. The frontend is deployed on Vercel, which provides automatic deployments from GitHub, global CDN distribution, and optimized SPA routing configuration.

## 2. Backend Architecture & API Design

The backend is a Flask REST API built with Python, providing a robust and scalable foundation for the application. The API is organized into logical modules: authentication endpoints (`/api/auth/register`, `/api/auth/login`) handle user account management with secure password hashing; chat endpoints (`/api/chat/send`, `/api/chat/transcribe`) process user queries and audio transcription; manual endpoints (`/api/manuals/upload`, `/api/manuals/delete`) manage PDF uploads and processing; and history endpoints (`/api/history`) retrieve chat conversations. The backend implements proper HTTP status codes, error handling with descriptive error messages, and CORS (Cross-Origin Resource Sharing) configuration to allow secure communication with the frontend. The API uses JSON for request/response serialization, making it language-agnostic and easy to integrate with different clients. The backend includes middleware for request validation, authentication verification, and error handling. The application implements rate limiting to prevent abuse and ensure fair resource allocation. The backend is deployed on Render, which provides a Linux container environment, automatic scaling, and persistent disk storage for the database.

## 3. Database Design & Data Persistence Strategy

The application uses SQLite as the primary database, chosen for its simplicity, reliability, and zero-configuration requirements. The database schema includes four main tables: the `users` table stores user account information including username, email, and hashed passwords (passwords are never stored in plain text); the `conversations` table stores metadata about each chat session including creation date, last updated timestamp, and conversation title; the `messages` table stores individual chat messages with content, sender information, timestamp, and associations to conversations; and the `manuals` table stores metadata about uploaded PDF files including filename, upload date, file size, and number of indexed chunks. The database implements proper indexing on frequently queried columns (user_id, conversation_id, timestamp) to ensure fast query performance. Data persistence is implemented using Render's persistent disk feature, which stores the database file at `/var/data` to ensure data survives application restarts and redeployments. A fallback mechanism is implemented that attempts to use the persistent disk but gracefully falls back to `/tmp` if permissions are denied, ensuring the application starts even if the persistent disk is temporarily unavailable. Database backups are recommended for production deployments to prevent data loss in case of infrastructure failures.

## 4. Deployment Infrastructure & DevOps Configuration

The application is deployed across multiple cloud platforms for optimal performance and reliability. The backend is hosted on Render, a modern cloud platform that provides free-tier hosting with 512MB RAM, automatic scaling, and persistent disk storage. Render is configured with environment variables for sensitive information (API keys, database paths) and automatic deployments triggered by GitHub pushes. The frontend is hosted on Vercel, a specialized platform for hosting Next.js and React applications, providing global CDN distribution, automatic HTTPS, and optimized SPA routing. Vercel automatically deploys the frontend whenever changes are pushed to the GitHub repository. Version control is managed through GitHub, which serves as the single source of truth for all code. GitHub is configured with secret scanning to prevent accidental exposure of API keys and sensitive credentials. The deployment process is fully automated: developers push code to GitHub, which triggers builds on both Render and Vercel, and the updated application is automatically deployed to production. Monitoring and logging are implemented to track application health, error rates, and performance metrics. The infrastructure is designed to be cost-effective, using free-tier services while maintaining production-grade reliability and performance. Disaster recovery procedures are documented, including database backup strategies and rollback procedures in case of deployment issues.
